在这次龙翔微信社区的项目中，因为对方将后台进行了封装，只以API形式提供数据，因此遇到了ajax传输数据的跨域 问题。总结一下ajax跨域的几种解决方案：

解决方案： 1、代理 前端将需要获取数据的Url请求发送给自己服务器的后台，自己服务器端将受到的请求转发给目标服务器（服务器端与服务器端是没有跨域的限制的，）,可以使用file_get_contents函数，再将获取到数据后返回给前端就可以了，不过上传图片貌似用这种方式不太妥当。

2、Jsonp 使用jsonp来获取数据，后台需要将返回的数据格式改为jsonp的数据格式（这里因为不是jsonp的格式报错了）

3、XHR2（IE10以下不支持） 添加信息头

总结： 在碰不了后台代码的情况下使用方法一为妙； 在可以修改后台代码的情况下方法三最方便，但同时要考虑兼容性的话可以使用方法二。

旧站迁移：中国大学生服务外包创新创业大赛决赛小结

服创大赛就此圆满结束了，同时也意味着我的暑假生活就此开始了（只有三天的暑假也是醉了），这次取得了全国二等奖成绩我已经很满足了。决赛主要展示能力和临场应变能力，同样也离不开那份底气，也就是对自己产品的自信能力。

在决赛期间小伙伴们发挥都很不错，没有出现什么大的失误，尽管我出场顺序抽了最后一签，但丝毫没影响大家士气，决赛进入答辩室后首先开始介绍我们的项目，中途评委老师问我们有没有成品，我们把手机拿给评委展示，评委针对我们产品提了相应的问题，都很好地解答了。主要问题还是问了和学校数据库接口方面问题还有项目的规划方面。

最后我对服创决赛提几点注意点：一是表现一定自信，讲解流畅；二是项目最后要有一个成品；三是一定要对自己的项目充分了解。

华为、同程活动报道

发表于 2015-08-15 | 分类于 活动报道

旧站迁移：中国计算机学会苏州会员活动——苏州华为研究院、同程旅游活动报道

今日有幸受奚老师之邀参加中国计算机学会苏州会员活动，主要参观访问了华为、同程旅游两家公司，今日之行，收益颇丰。 ##一、参观华为研究所 到达了园区科技创业园后，首先映入眼帘的是华为研究所气派的大楼，走进华为究所，完成签到后，便在华为工作人员的带领下进入展厅，我们逐一参观了华为科技在各个领域的成果，同时了解了华为公司的业绩，不得不令人赞叹华为公司其业务范围之广，科技成就之高，其业务涵盖了城市、铁路、电网、能源、银行、医院、电视媒体、酒店、零售等各行各业，其间给我印象最深的是一个宣传短片，短片中并没有刻意强调华为的技术有多好，而是通过一个故事的形式展示了华为技术的重要性。故事讲述了一次马拉松比赛的过程中发现了一枚定时炸弹，之后从视频监控，视频会议拆弹，人脸识别，无人机抓捕，到第二枚炸弹的信息屏蔽，无不展现了华为科技的技术含量之高以及其重要性。之后我又开始了大约半小时的视频会议，会议中华为公司的相关人员讲述了华为企业如何与高校进行对接，同时提出了“华为网院”的平台，作为高校师生与企业进行对接的桥梁。

华为经济增长指数

二、参观同程网络科技股份有限公司

参观了华为研究所后，我们又来到了同程网络科技股份有限公司，在同程公司CTO张海龙的带领下首先参观了一下同程公司，了解了同程公司的发展历程，令人惊叹的是从一个当初只有两个人的小公司发展到如今拥有几千员工的大公司。之后张总由于我们分享了公司一路发展过来所经历的艰辛与喜悦，以及他在公司发展这么多年的个人经验，每一句都是作为一个企业家的肺腑之言，之后张总又与我们讲述他对于当今企业对大学生人才要求的看法与理解，对我的启发很大，张总说，对于当初我们还是一个小企业的时候，我们可能需要的是那种应用型的人才，上来就能办事的那种人，现在企业规模扩大了，我们如今需要的是哪些有基础有底蕴的人，并不是那种掌握了某种开发语言的人，这些其实不重要了，因为企业可以教你，甚至教的比学校里面更好。

服务外包大赛初赛

发表于 2015-08-05 | 分类于 参赛感言

中国大学生服务外包创新创业大赛初赛小结

这次初赛能够晋级我感到很高兴，首先我要感谢我们的指导老师对我们项目细心指导，然后我想要感谢我们团队的每一位成员，没有他们的辛苦付出与努力是不会取得如今的成果的，最后同样要感谢在座的每一位老师，感谢你们对我们的作品提出的每一份宝贵的意见。

接着我想要总结一下我们的团队。一方面我们团队的成员都能够以较高的效率完成所布置的任务，个体素质这在一个团队里面是很重要的，但相应的，团队成员之间缺乏一定的交流和团体的思想，往往只顾着完成自己眼下的工作，就好比前端只顾着把页面完成，却不管与后台数据交互，或者后台的数据格式是怎样的不能明确传递给前台，总之我希望的是团队之间在开发的时候能够增进交流。另一方面，在职能上，我们团队有三位成员是开发能人员（另外两名分别负责文档与市场），我觉得三名开发人员在五人团队里能够发挥最大效率，但在三人里面有两名前端，一名后台开发人员，结构上多少存在点问题，这和我们这个项目的先天性不足（开发语言不是所学语言）有关，虽然前期也尝试后台分配两人，但是并没能够很好的运转。所以后台这一块较为吃紧。

关于项目再说说我的看法，这个项目从一开始我们决定了以后，就开始进行需求分析，然后与商学院同学合作，开始市场调研，同时与教务处老师进行交流获取相应数据，之后开始原型设计，系统设计，每一步都抓得很紧，同时在项目期间也经历过挫折，也正是这样促使了我们进行界面的重构，系统的界面风格发生了很大的改变，同样也开始趋于稳定，就这样系统的2.0版本诞生了。

1.0与2.0
